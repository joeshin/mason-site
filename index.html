Introduction

Mason is a powerful Perl-based templating system for generating HTML or other dynamic content.

Unlike many templating systems, Mason does not invent an alternate, "easier" syntax for templates. It simply lays a friendly templating syntax over the versatile and well-documented Perl language.

Mason has been around for fourteen years (originally as HTML::Mason) and has powered thousands of web sites large and small, including Amazon.com and Salon.com. Mason 2 was launched in 2011.

Poet is a modern companion web framework for Mason. It uses PSGI/Plack for server integration and a selection of best-of-breed CPAN modules for caching, logging and configuration.

If you are already using another Perl web framework, such as Catalyst or Dancer, Mason can also be used as its templating layer.

Quick Start

Install cpanm if necessary. Then run 
% cpanm -S --notest Poet

This will install Poet and all prereqs, including Mason.
Omit the "-S" if you don't have root, in which case cpanminus will install Poet and prereqs into ~/perl5.
Omit the "--notest" if you want to run all the installation tests (will take about x4 as long).

To create your initial environment: 
% poet new MyApp
my_app/.poet_root
my_app/bin/app.psgi
my_app/bin/get.pl
...
Now run 'my_app/bin/run.pl' to start your server.
Then start your server: 
% my_app/bin/run.pl
Running plackup --Reload ... --env development --port 5000
Watching ... for file updates.
HTTP::Server::PSGI: Accepting connections at http://0:5000/
